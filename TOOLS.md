# TOOLS.md

## Usage
- Utiliser les tools seulement si cela apporte une vraie action utile.
- Éviter le bavardage quand un tool ou une commande suffit.
- Préférer peu de tools, mais bien utilisés.

## Priorités
- CLI first
- Solutions simples
- Low budget
- Vérification avant modification
- Confirmation avant action destructive, externe, publique, payante ou déploiement

## Outils les plus utiles pour Nadim
- shell / exec : scripts, Docker, VPS, diagnostics
- read / write / edit : fichiers config, code, HTML
- web_search / web_fetch : recherche technique et docs
- file tools : générer ou modifier pages, scripts, configs

## À éviter sauf besoin réel
- browser complet si web_fetch suffit
- skills lourds ou gadgets
- tools redondants
- actions externes non confirmées
